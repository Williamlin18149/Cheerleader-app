# 04 — 功能模組

以下對應 `phase==="app"` 時的主要 UI 與 state（實作以 `index.html` 為準）。

## 我的選擇（`tab === "browse"`）

- 依 `lf`（聯盟）、`tf`（球隊）、`nf`（國籍）、`q`（關鍵字）、`multi`（雙棲以上）篩選 `members` 得到 `filtered`。
- `browseFriendFilter` 可改為只看某位房友的 `faves` 子集合（仍可用上方球隊篩選）。
- 卡片點擊切換「我的」`myFaves`（寫回 RTDB 自己節點）。
- 卡片上可顯示「誰也選了此人」（顏色圓點對照 `colorMap`）。

## 共同支持（`tab === "overlap"`）

### 計入對象

- `overlapRoomPick`：以暱稱為 key，勾選是否計入（預設全勾；可全選／全不選）。
- 僅 **勾選為 true** 的暱稱會進入計算。

### 顯示方式（`overlapListMode`）

| 值 | 意義 |
|----|------|
| `everyone`（預設） | **交集**：只顯示「每一位計入成員的 `faves` 都包含」的啦啦隊員 id。僅勾一人時，即為該人完整名單。 |
| `anyone` | **聯集**：只要有一位計入成員選過就列出；`cnt` 為幾人選過，並列出 `likers`。 |

### 篩選

- 與 browse 共用 `lf` / `tf` / `nf`（聯盟改變時會重置球隊為 `all` 的邏輯與 browse 一致處需以程式為準）。

### 空狀態文案

- 房間人數不足、無人勾選計入、或篩選後無結果時，有不同提示（見程式內 `empty` 區塊）。

## 房間成員（`tab === "members"`）

- 列出 `roomData` 的 keys（暱稱）與每人選取數量預覽。
- 點擊開啟 modal，顯示該人完整 `faves` 對應的成員卡片／列。

## 管理後台（`phase === "admin"`）

- 先輸入 `ADMIN_PASSWORD`（內嵌常數）進入面板。
- 兩大區塊（`adminSection`）：
  - **房間管理**：列出房間、成員列、清空選擇／刪除成員／清空房間等（受 RTDB 規則影響）。
  - **啦啦隊名單編輯**：表格編輯、表單新增／修改、寫回 `config/members` 並更新本地 `members` state。

## Session 還原

- `localStorage` key：`SESSION_KEY`（`cheer_session_v2`）存 `{ name, room }`。
- 開機嘗試自動 `once` 讀取遠端節點確認房間存在後進入 `app`。

## 登出（`leave`）

- 清除 session、關閉 listener、重置多項 UI state（含 browse／overlap 相關篩選與 `overlapListMode` 等，以程式為準）。
